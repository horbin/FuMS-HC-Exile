BaseServer

 

File:

        FuMS\Themes\BaseServer.sqf

 

Description:

            This file contains configuration information that impacts core FuMS functionality and details that apply across all themes and missions.  This file also controls what themes should be loaded when FuMS initializes.

 

Sections:

1)   Core Configuration

a)    Determines FuMS and Admin tool behavior.

b)    Parameters

i)      FuMS ON SERVER: BOOL: false

(1)   true = enables FuMS to run missions on the main server. Enabling this option can result in serious performance degradation on the server. This option is primarily provided to allow people who do not have an HC to run and test FuMS before going through learning how to set up the Headless Client.

(2)   false = FuMS will not spawn missions on the server.  All missions and AI are controlled by the Headless Clients

 

ii)      DATABASE CONFIG: STRING: "testpath"

(1)   Place holder for future MySQL/REDIS data persistence and configuration support.

 

iii)    ADMIN CONTROLS: BOOL: true

(1)  true = enables FuMS administrative menu system. This menu system provides basic controls to allow designated admins to start/stop missions

(2)  false = disables FuMS admin menu system.

(3)  See Docs folder for further details on Admin Controls.

 

iv)    MIN FRAMERATE: SCALAR : 15

(1)  FuMS will monitor the server FPS. When the FPS falls below the value specified, FuMS will suspend mission creation and script communication with the Headless Clients until the FPS returns above the set value.

(2)  90% of the server load from FuMS occurs during HC connection, due to the data transfer between server and HC. Because of this fact, the frame rate restrictions no longer apply to an 'operating' HC.

 

c)    Notes

i)      \FuMS\HC\Util\GetWorldInfo.sqf

(1)   FuMS will work on any Arma2 or Arma3 map that has a valid map definitions file (ie is a working map).  See the above file for details.

(2)   If you are adding a new map, OR need to add exclusion areas, such as spawn points, you may need to modify or add to data in this file.

(3)   All stock Themes that come with FuMS are designed to operate on any map with no additional setup required.

(4)   Of course, FuMS is an Admin's mission tool, so it IS highly recommended you tailor each theme to your server's loot/player experience level. By default FuMS loot is fairly generic and the AI mediocre marksman.

 

2)   Active Themes

a)    List all the themes that FuMS should load and control.  When more than one theme is being run by FuMS, ensure each is separated by a comma. Also ensure the last theme in the list contains NO trailing comma.

b)    Commenting out an item will DISABLE the theme (ie Turn it off)

c)    Parameters

i)      ["Test", -1],

(a)  "Theme Name" : STRING : "Test"

(i)    Theme name is NOT case sensitive (it will be converted to caps)

(ii)   Should be a single word or continuous string with no special characters

(iii)  Must match a folder in \FuMS\Themes\

 

(b)  HC Control: SCALAR : -1

(i)    Value controls where FuMS will control the theme

(ii)   -1 = Theme will run on server (if server operation enabled), and ALL HC's that connect to the server

(iii)  0 = Theme will run on server (if server operation enabled)

(iv) 1,2,3 ... = Theme will only run on the 1st, 2nd, 3rd... HC to connect to the server

 

d)    Notes

i)      When a theme starts, it instantiates its own mission control loop that is independent of all other themes.

ii)     Themes that start on the server and/or HC's each have their own control loop and operate independently.

iii)    A theme CAN be listed multiple times, each theme running independently.

      EX: ["SEM",-1], ["SEM",-1],["SEM",-1] will run 3 instances of the SEM theme, resulting in 3 active missions, 1 under each theme, running at all times.

 

 

3)   Radio Chatter Configuration

--These options are non-functional for Exile--

a)    Provides for turning Radio chatter support ON/OFF for all of FUMS, as well as some other global settings

b)    Parameters

i)      ENABLE RADIOCHATTER : BOOL : true

(1)  true = turns on chatter system, allowing AI radio conversations to be seen in the upper left corner of the display

(2)  false = turns off all radio chatter support options (to include reinforcements (see Triggers.txt)

 

ii)     ENABLE RADIOAUDIO: BOOL: false

(1)  true = no longer valid. FuMS radio audio currently disabled

(2)  false = turns off sound effects from AI radio chatter

 

iii)    RADIO REQUIRED: BOOL: true

(1)  true = player must have a radio on his belt to see/hear radio chatter messages

(2)  false = players will hear/see messages at all times (when within range)

 

iv)   RADIOFOLLOW THEME: BOOL: true

(1)  true = configuration in each theme's ThemeData.sqf will be referenced for the theme's radio behavior

(2)  false = all FuMS themes will follow setting specified in this section

 

v)    ENABLE AI CHATTER: BOOL: true

(1)  true = turns on random radio chatter between AI and their base of operations

(2)  false = turns off random chatter, death messages will still occur.

 

vi)   RADIO RANGE: SCALAR: 800

(1)  Sets the range, in meters, that AI radio chatter can be heard.

(2)  This distance is measured from a player's location to the mission centers

(3)  chatter from each theme's BASE Operations always has unlimited range.

 

c)    Notes:

i)      None.

 

4)   Soldier Defaults

a)    Provides basic load-out and logic behavior, as well as control over AI specific equipment.

b)    Parameters

i)      RIFLE MAGS: SCALAR: 3

(1)  AI created will always have at least 3 rifle magazines appropriate to their weapon upon AI creation.

 

ii)     PISTOL MAGS: SCALAR: 3

(1)  AI created will always have at least 3 pistol magazines appropriate to their weapon upon AI creation.

 

iii)    VCOM DRIVING: BOOL: true

(1)  true = enables Genesis92x's VCOM_Driving V1.01 for AI in vehicles

(2)  false = AI will utilize default Arma3 driving logic

 

iv)   SKILLS ARRAY: ARRAY:

(1)  Values specified here OVERRIDE any value set in any other soldier configuration file

(2)  Setting a value to zero, permits settings from other FuMS files to take effect

(3)  Default values : (value within the comment is the default set by SoldierData.sqf and what will be used if the value is set to zero in this file.)

                 [

                  .8, // aimingAccuracy .05 : target lead, bullet drop, recoil

                  .9,        // aimingShake .9 : how steady AI can hold a weapon

                  .5,        // aimingSpeed .1 : how quick AI can rotate and stabilize its aim and shoot.

                  .9,        // spotDistance .5 : affects ability to spot visually and audibly and the accuracy of the information

                  .8,        // spotTime .5 : affects how quick AI reacts to death, damage or observing an enemy.

                  .9,        // courage .1 : affects unit's subordinates morale

                        .5,        // reloadSpeed .5 :affects delay between weapon switching and reloading

                  .8         // commanding .5 : how quickly recognized targets are shared with the AI's group.

                  ],

 

v)    SOLDIER ONLY GEAR: ARRAY

(1)  Items listed in these lists will be deleted upon AI death, preventing players from having access to the equipment.

(2)  Default values:

        [

            [], //Uniforms

            [], // Vests

            [], // Backpacks.

            [], // Helmets

            ["launch_RPG32_F","launch_I_Titan_F"], // Weapons

            ["RPG32_HE_F","RPG32_F","Titan_AA"], // Magazines

            [] // Items

        ]

 

(3)  The above defaults ensure the stock AT and AA weapons used by the AI are not available to players.

(4)  Other custom equipment can be placed here to permit admins to tailor the look and capability of the AI on his server.

(5)  This is a DELETION list. Equipment still needs to be placed on the AI in the appropriate SoldierData.sqf file.

(6)  This array only supports individual "String" names. Other array objects defined within FuMS is not recognized by this array, so items must listed individually.

 

vi)   UAV AND UGV's: ARRAY

(1)  List all UAV and UGV class names that FuMS should recognize as AI controlled.

(2)  UV 'side' is determined by the prefix on the class name (I=Resistance, O=East, B=West).

(3)  UV 'side' is NOT determined by the driver 'side' defined in the mission file.

 

c)    Notes:

i)      See GlobalSoldierData.sqf and SoldierData.sqf for detailed AI configuration.

ii)     See each Theme's 'ThemeData.sqf' to control which SoldierData file the theme uses when building AI.

 

 

5)   Loot Defaults

a)    Provides global configuration of loot rewards and sets restrictions on player vehicles

b)    Parameters

i)      DELETE LOOT DELAY : SCALAR: 20 (minutes)

(1)  Establishes a timer that starts after mission cleanup occurs.

(2)  When timer expires loot boxes created by the theme's last expired mission will be deleted.

 

ii)     SMOKE BOX OPTIONS: ARRAY

(1)  These setting allow for the configuration of 'smoke' signals to aid players in locating loot boxes

(2)  Default settings:

[

            true, // create smoke

            100, // start smoke when a player gets this distance from the box

            ["Red"], // color of smoke

            1 // duration of smoke in MINUTES

]

(3)  Setting the distance to 0 will make the distance unlimited

(4)  Smoke can also be defined in an array.

(a)  ["Red","White","Blue"], ["Green","White"]

 

iii)    VEHICLE PERSISTANCE: BOOL : true

(1)  true = vehicles occupied by players are sellable and will persist through server reset.

(2)  false = vehicles NOT sellable and will vanish at server reset.

(3)  FuMS provides a message to all players entering a FuMS created vehicle of its potential temporary status. This message goes away for vehicles that remain through a server reset.

 

iv)   CRATE OPTIONS: ARRAY: [B_supplyCrate_F","O_supplyCrate_F","I_supplyCrate_F","CargoNet_01_box_F"]

(1)  Within LootData.sqf and GlobalLootData.sqf the type of box created for loot is defined by each loot type.

(2)  When the key word "Random" is used in these files, the above array is referenced for the selection of loot container.

 

v)    PROHIBITED VEHICLES: ARRAY: ["I_UGV_01_rcws_F","B_G_Offroad_01_armed_F"]

(1)  Vehicles listed in this array will not be usable by players

(2)  Players will receive a notification of this fact when attempting to enter a vehicle on this list.

(3)  Vehicles on this list will function normally for AI.

 

vi)   VEHICLE AMMO: BOOL : true

(1)  true = vehicle ammo set to zero when a player FIRST takes possession of a vehicle from AI.

(2)  false = vehicle ammo remains intact when players capture a vehicle from AI.

(3)  This flag does NOT impact ammo in player owned vehicles. It only applies to vehicles spawned by FuMS before they become player controlled.
